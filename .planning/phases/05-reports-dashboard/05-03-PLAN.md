<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → PDF views | GET com params de filtro idênticos às views web; resposta é download de arquivo |
| view → pdf builder → buffer | Dados do service viram bytes em memória; sem escrita em disco |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Information Disclosure | GastosPDFView / RequisicoesPDFView | mitigate | `RelatorioRequiredMixin` bloqueia solicitante (403); os PDFs usam os MESMOS filtros via `_parse_filtros`/service — não aceitam unidade arbitrária para roles não autorizados (RESEARCH §Known Threat Patterns — PDF com dados de outra unidade) |
| T-05-08 | Tampering | _parse_filtros nos endpoints PDF | mitigate | Reutiliza o mesmo helper validado (strptime + try/except); datas inválidas caem no default |
| T-05-09 | Denial of Service | PDF em disco | mitigate | Geração 100% em `BytesIO` in-memory; nunca `open(path,'wb')` (seguro em Docker/multi-processo) |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Nenhum pacote novo — `reportlab` já em requirements.txt (RESEARCH §Package Legitimacy Audit) |
</threat_model>
