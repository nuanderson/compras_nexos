<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → password reset view | Untrusted email submitted — no authentication required |
| Password reset email → browser | HMAC token in URL — must expire, must be single-use |
| Test suite → test database | Tests use isolated test DB — no prod data access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Elevation of Privilege | Password reset token reuse | mitigate | Django invalidates the reset token after first successful use via `PasswordResetConfirmView`. `PASSWORD_RESET_TIMEOUT` defaults to 3 days. No additional code needed. |
| T-01-10 | Information Disclosure | Password reset — user enumeration | mitigate | Django `PasswordResetView` always shows the "done" page regardless of whether the email exists (confirmed by RESEARCH.md and UI-SPEC copywriting: "Se esse e-mail estiver cadastrado..."). No information leakage. |
| T-01-11 | Spoofing | Password reset email link | mitigate | Reset URL uses HMAC-based token tied to user's password hash. If password changes, all prior tokens are invalidated automatically. |
| T-01-SC | Tampering | pip package installs | mitigate | All packages inherited from Plan 01 — no new packages added in this plan. Legitimacy audit already passed. |
</threat_model>
