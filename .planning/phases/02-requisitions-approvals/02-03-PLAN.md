## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → views aprovacoes (Gestor) | Gestor autenticado dispara aprovação/reprovação (entrada e identidade não confiáveis) |
| view → services | Enforcement de estado, alçada e ownership de unidade |
| services → SES (e-mail) | Efeito colateral externo transacional (on_commit) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | Gestor aprovando requisição de outra unidade | mitigate | Queryset filtra `unidade=request.user.default_unit`; views de ação verificam `req.unidade == request.user.default_unit` antes de chamar o service (senão PermissionDenied) |
| T-03-02 | Spoofing | Solicitante acessando fila/ações de Gestor via URL | mitigate | `GestorRequiredMixin` em todas as views de aprovação |
| T-03-03 | Tampering | Reprovação sem motivo (bypass do required HTML) | mitigate | `ReprovaForm.clean_motivo` + validação no service `reprovar_requisicao` (defense in depth, APROV-05) |
| T-03-04 | Tampering | Aprovação dupla concorrente | mitigate | `select_for_update()` no service (Plano 02) + retorno 409 quando estado não aceita ação (Armadilha 4) |
| T-03-05 | Information Disclosure | E-mail vazando para Gestores de outra unidade | mitigate | `_notificar_gestores` filtra `default_unit=req.unidade, is_active=True` |
| T-03-06 | Cross-Site Request Forgery | Modal de reprovação (form HTMX injetado) | mitigate | `{% csrf_token %}` no form do modal + meta csrf-token + handler em base.html (CR-04) |
| T-03-SC | Tampering | npm/pip installs | accept | Nenhum pacote novo nesta fase (RESEARCH.md §Auditoria de Legitimidade — não aplicável) |
</threat_model>
