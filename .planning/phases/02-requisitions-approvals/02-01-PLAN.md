| admin/shell → ConfiguracaoAlcada singleton | Tentativa de criar múltiplas linhas de alçada quebraria a lógica de singleton `obter()` (pk=1) |
| admin → AprovacaoLog (trilha de auditoria) | Logs de aprovação devem ser imutáveis para preservar a integridade da auditoria (REQ-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Elevation of Privilege | Acesso não autorizado ao Django admin de ConfiguracaoAlcada / AprovacaoLog | mitigate | Django admin exige `is_staff` por padrão (`admin.site.login` rejeita não-staff); apenas usuários staff criados pelo Admin alcançam essas telas (ASVS L1 V1.2) |
| T-02-01-02 | Tampering | Criação de 2ª ConfiguracaoAlcada via admin/shell, burlando o singleton `obter()` (pk=1) | mitigate | Override de `has_add_permission` retorna `not ConfiguracaoAlcada.objects.exists()`, bloqueando a inserção de uma segunda linha pelo admin; `has_delete_permission` retorna `False` para impedir remoção e recriação fora do pk=1 |
| T-02-01-03 | Repudiation | Edição/exclusão de AprovacaoLog corrompendo a trilha de auditoria (REQ-03) | mitigate | `AprovacaoLogAdmin` define `readonly_fields` em todos os campos, `has_add_permission=False` e `has_delete_permission=False` — log estritamente imutável via admin |
| T-02-01-SC | Tampering | npm/pip installs | accept | Nenhum pacote novo instalado neste plano (ver RESEARCH.md §Auditoria de Legitimidade — não aplicável) |
</threat_model>
